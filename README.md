# Лендинг врача-ревматолога Мовсесян Анастасии Арменовны

Это стартовая структура статического сайта для будущего лендинга врача-ревматолога Мовсесян Анастасии Арменовны. На текущем этапе подготовлены файловая архитектура, базовые HTML-страницы, CSS-слои, JS-заготовки и SEO-файлы без финального дизайна, контента, backend-интеграций и оплаты.

## Созданные страницы

- `index.html` — главная страница с пустыми секциями для будущей верстки.
- `questionnaire.html` — страница анкеты перед онлайн-консультацией с placeholder-контейнером формы.
- `success.html` — страница успешной отправки анкеты с временным текстом.

## Структура стилей

Основная точка подключения стилей — `assets/styles/main.css`.

Стили разделены по назначению:

- `assets/styles/base/` — reset, переменные, типографика, глобальные стили и утилиты.
- `assets/styles/components/` — будущие переиспользуемые компоненты: кнопки, карточки, бейджи, аккордеон, поля ввода и анимации.
- `assets/styles/layout/` — стили секций и крупных блоков лендинга.
- `assets/styles/pages/` — стили отдельных страниц.
- `assets/styles/responsive/media.css` — отдельный файл для адаптивных стилей.

## Изображения и файлы

- Фото врача добавлять в `assets/images/doctor/`.
- Иконки добавлять в `assets/images/icons/`.
- Фоновые изображения добавлять в `assets/images/backgrounds/`.
- Временные изображения и заглушки добавлять в `assets/images/placeholders/`.
- Документы добавлять в `assets/docs/` только при необходимости. Реальные медицинские файлы пациентов не должны храниться на сайте.
- Шрифты добавлять в `assets/fonts/`.

## JavaScript

JS-файлы лежат в `assets/js/` и разделены по задачам:

- `main.js` — общий инициализатор.
- `menu.js` — будущая логика мобильного меню.
- `accordion.js` — будущая логика FAQ-аккордеона.
- `form.js` — будущая обработка анкеты и место для подключения Telegram/email-интеграции.
- `animations.js` — будущие анимации при скролле.

## Интеграции и аналитика

- Telegram/email-интеграцию позже подключать через `assets/js/form.js` и/или отдельный backend/serverless-слой. Не указывать реальные токены, email-пароли и медицинские файлы в публичном коде.
- Яндекс.Метрику позже вставлять перед закрывающим тегом `</head>` или перед `</body>` в HTML-файлах, в зависимости от актуальных рекомендаций Яндекса.
