# Лендинг врача-ревматолога Мовсесян Анастасии Арменовны

Версия 1.0 — статический MVP персонального сайта врача-ревматолога в светлом медицинском минималистичном стиле.

## Что реализовано

- Полноценная главная страница `index.html` с hero-блоком, фото врача, секциями «О враче», «С чем обратиться», «Кому подходит», «Как проходит консультация», подготовкой, услугами, отзывами-placeholder, публикациями-placeholder, FAQ, юридическим дисклеймером, финальным CTA и footer.
- Страница анкеты `questionnaire.html` с формой, крупными чекбоксами симптомов, полями контактов и ссылкой на облако для анализов/снимков/выписок.
- Страница успешной отправки `success.html` с понятным сообщением и кнопками возврата/Telegram.
- Рабочее мобильное меню в `assets/js/menu.js`.
- FAQ-аккордеон в `assets/js/accordion.js`.
- Имитация отправки анкеты в `assets/js/form.js`: данные не отправляются на сервер, после успешной клиентской проверки происходит переход на `success.html`.
- Мягкие scroll-анимации в `assets/js/animations.js`.
- Адаптив для desktop/tablet/mobile в `assets/styles/responsive/media.css`.
- SEO title/description и Open Graph на страницах, а также комментарии для будущего подключения Яндекс.Метрики.

## Важно по интеграциям

На текущем этапе не подключены:

- реальная отправка данных в Telegram/email;
- backend или serverless-обработчик формы;
- реальная оплата;
- хранение медицинских файлов на сайте.

Основной сценарий для документов пациента — вставить ссылку на Яндекс.Диск или другое облако в анкету. Реальная отправка уведомлений в Telegram/email будет подключена следующим этапом через backend/serverless-слой без хранения медицинских файлов на сайте и без публикации секретов в репозитории.

## Структура стилей

Основная точка подключения стилей — `assets/styles/main.css`.

Стили разделены по назначению:

- `assets/styles/base/` — reset, переменные, типографика, глобальные стили и утилиты.
- `assets/styles/components/` — переиспользуемые компоненты: кнопки, карточки, бейджи, аккордеон, поля ввода и анимации.
- `assets/styles/layout/` — стили секций и крупных блоков лендинга.
- `assets/styles/pages/` — стили отдельных страниц.
- `assets/styles/responsive/media.css` — адаптивные стили.

## Изображения и файлы

- Фото врача используется из `assets/images/doctor/movsesyan.png`.
- Иконки добавлять в `assets/images/icons/`.
- Фоновые изображения добавлять в `assets/images/backgrounds/`.
- Временные изображения и заглушки добавлять в `assets/images/placeholders/`.
- Реальные медицинские файлы пациентов не должны храниться на сайте.

## Проверка локально

Откройте `index.html`, `questionnaire.html` и `success.html` в браузере или запустите простой статический сервер:

```bash
python3 -m http.server 8000
```

Затем проверьте:

- открытие/закрытие мобильного меню;
- раскрытие FAQ;
- переходы CTA на `questionnaire.html`;
- имитацию отправки формы и переход на `success.html`;
- адаптивные состояния в DevTools.
